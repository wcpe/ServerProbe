# 数据呈现与对接

> ⚠️ 本页为设计阶段文档,描述目标行为,功能尚未实现。

ServerProbe 计划提供**四个呈现 / 对接通道**(对应 PRD FR4)。同一份聚合结果向多个出口共享,互不重复采集。

> ⚠️ 所有通道与配置项均为 🚧 规划中,功能尚未实现;下方配置仅为占位示意,以最终实现为准。

| 通道 | 优先级 | 适用场景 |
|---|---|---|
| 游戏内命令 | P0 | 即时巡检、单次定位"开服慢" |
| Prometheus `/metrics` + Grafana | P1 | 长期监控、看板、报警 |
| Web 面板 | P2 | 启动画像详情、历史趋势、(可选)火焰图 |
| 本地文件历史落盘 | P0/P1 | 历史对比、事故回溯、数据沉淀(不依赖数据库) |

---

## 一、游戏内命令(P0)

最直接的出口,无需任何外部组件。入口 `/probe`,子命令 `health|startup|tps|gc|world|proxy`,权限受控。

适用场景:服主临时排查、运维快速确认当前状态、开发者看自己插件 enable 耗时。

完整命令、示意输出、权限节点见 [命令与权限](Commands.md)。

---

## 二、Prometheus `/metrics` + Grafana(P1)

把指标暴露为 Prometheus 抓取端点,接入 Grafana 做长期看板与告警。

- Bukkit 与 BungeeCord **各一套** `/metrics`(代理端导出网络/子服健康类指标,无 TPS/MSPT/世界)。
- 端口可配;端点需**鉴权 + 绑定地址限制**,输出不泄露路径 / token。

配置占位(🚧 规划中,字段名以实现为准):

```yaml
# config.yml(示意)
prometheus:
  enabled: false          # 默认关闭
  bind: 127.0.0.1         # 绑定地址,限制暴露面
  port: 9940              # 抓取端口,可配
  auth-token: ""          # 鉴权 token(留空=拒绝外部访问)
```

Prometheus 抓取配置(示意):

```yaml
scrape_configs:
  - job_name: serverprobe
    static_configs:
      - targets: ['127.0.0.1:9940']
```

适用场景:多服 / 长期监控、需要历史趋势图与阈值报警、已有 Grafana 体系。

> 依赖策略(已敲定):**允许按需引入轻量依赖,但每个新依赖需逐个确认**。Prometheus 客户端依赖据此评估,按依赖管理规范逐个确认后引入,见 [常见问题FAQ](FAQ.md)。

## 三、Web 面板(P2)

内置 Web 面板,用于查看启动画像详情、历史趋势,以及(可选)火焰图。

- 需**鉴权 + 绑定地址**(安全要求,不裸暴露)。
- 优先级 P2,排在命令 / Prometheus / 落盘之后。

配置占位(🚧 规划中):

```yaml
web:
  enabled: false
  bind: 127.0.0.1
  port: 8980
  auth: ""                # 访问凭据
```

适用场景:需要图形化查看启动画像细节与历史趋势,不想自建 Grafana。

> 依赖策略(已敲定):**允许按需引入轻量依赖,但每个新依赖需逐个确认**。内嵌 Web 服务依赖(如 Javalin / NanoHTTPD)据此评估,逐个确认后引入。

## 四、本地文件历史落盘 + 开放接口(P0/P1)

探针**不访问数据库**,把数据沉淀到**本地文件**,支撑"与上次 / 基线对比"和事故回溯。

落盘内容:
- **启动画像**:每份启动画像写为**一个 JSON 文件**(`StartupProfile`)。
- **指标历史**:**聚合后**以 **JSONL 行式追加**(`MetricHistory`),每行一条记录;写入频率受控。

设计约束:
- **原子写入**:先写临时文件再 `rename`,避免半截文件;可配**滚动**与**保留**策略控制磁盘占用。
- **异步写入**,严禁主线程阻塞磁盘 IO;高频指标先进环形缓冲,聚合后才落盘。
- **不依赖数据库**:无需任何外部数据源或连接配置。

配置占位(🚧 规划中):

```yaml
storage:
  enabled: true
  # 存储目录 / 文件滚动 / 保留策略(占位)
  retention-days: 30      # 历史保留策略
```

适用场景:需要历史对比定位"这次比上次慢多少"、长期数据沉淀与事故复盘。多服共存时的实例标识已敲定:**`serverId` 由实例自动生成,可在配置中以 `server-name` 覆盖为自定义值,本地文件按实例分目录存放**,互不混淆。

数据模型草案见 [指标说明](Metrics.md) 与 [架构文档](../ARCHITECTURE.md)。

### 开放接口

探针对外开放三层接口,既供查询也预留扩展:

- **① 读取 API**:`api` 模块提供**只读数据访问**契约,第三方插件经 TabooLib 服务获取启动画像与指标历史。
- **② 存储 SPI**:存储后端通过 SPI 抽象,**默认且唯一内置 = 本地文件**;预留扩展点供第三方自接 DB / 远程存储,但插件本身不内置、不依赖。
- **③ 导出端点**:对外导出走 Prometheus `/metrics` 与 Web 面板(见上文第二、三节)。

## 通道与场景速查

| 你的需求 | 推荐通道 |
|---|---|
| 现在就想看一眼服务器状态 | 游戏内命令 `/probe health` |
| 这次开服为啥这么慢 | 游戏内命令 `/probe startup` + 历史文件对比 |
| 做长期看板 + 报警 | Prometheus + Grafana |
| 图形化看启动画像细节 | Web 面板 |
| 事故回溯 / 历史趋势 | 本地文件历史落盘 |
