# ADR-9：代理端暂不联动汇总，各端独立

## 状态
已接受

## 背景
现状追认：本决策早于本次 SDD 逆向即已成立，依据 docs/ARCHITECTURE.md §11 补记为独立 ADR。

ServerProbe 覆盖 Bukkit 系后端与 BungeeCord 代理端。一个自然的扩展方向是让代理端联动后端、做跨服指标汇总。需要决定首期是否引入这种跨端联动。

## 决策
代理端暂不与后端联动汇总，各端独立采集与展示。

## 理由
- 先打基础、降低耦合：各端独立采集/展示，避免过早引入跨端协议与状态同步的复杂度。
- 联动作为后续可选增强，而非首期必做（PRD §12）。
- 跨服聚合需求可由外部系统经开放接口（只读读取 API）完成，无需在插件内置跨端汇总（PRD §12 多实例标识）。

## 后果
- 正面：模块解耦、实现简单、降低耦合面；各端可独立演进。
- 负面/约束：首期不提供开箱即用的跨服聚合视图，跨服汇总须由外部系统经开放接口自行实现；本地文件按实例分目录存放，多实例标识用自动生成 ID + 可配 `server-name` 覆盖。
- 约束：联动汇总作为后续可选增强保留，引入时不得破坏各端独立采集的基础。

## 备选方案
- **Porticus 跨服汇总**：被否决。首期优先打基础、降耦合，不引入 Porticus；跨端联动作为后续可选增强。
