# ADR-8：运行期火焰图不自研并用 spark，启动期火焰图自研

## 状态
已接受

## 背景
现状追认：本决策早于本次 SDD 逆向即已成立，依据 docs/ARCHITECTURE.md §11 补记为独立 ADR。

ServerProbe 的首要场景是"开服慢剖析"，需要对启动期（premain 窗口）的"无日志卡顿"热点给出可视化（火焰图）。同时运维场景也涉及运行期 CPU 热点分析。需要决定：CPU 热点/火焰图能力，哪些自研、哪些复用既有成熟工具。

## 决策
**运行期** CPU 热点/火焰图不自研，建议并用 [spark](https://spark.lucko.me)；**启动期（premain 窗口）**火焰图自研（M5，`/probe flamegraph`）。

## 理由
- spark 成熟、是社区标准，运行期深度 CPU 火焰图复用它即可，不重复造轮子；ServerProbe 专注指标监控 + 启动剖析（PRD §12）。
- 但 spark 难以介入 premain 阶段，而"启动期"恰是本项目的首要场景，spark 覆盖不到 premain 窗口。
- 因此自研一个轻量、启动期专用的折叠栈火焰图：由可选 premain agent 采集的多线程折叠栈/时间线，经 `/probe flamegraph` 生成自包含 HTML（CSS/JS 全内联、无 CDN）到 `data/flamegraph/`，含真正多层多线程火焰图 + 按区间嵌套关系分泳道的时间线。
- 两者并用、互补不冲突：启动期自研、运行期 spark。

## 后果
- 正面：启动期"无日志卡顿"热点首次可量化、可视化，补齐了 spark 的盲区；运行期不重复造轮子，降低维护面。
- 负面/约束：自研火焰图仅覆盖启动窗口；运行期深度 CPU 分析需用户自行部署 spark。启动火焰图能力依附于可选 premain agent（见 ADR-11），不启用 agent 则无此输出。
- 约束：运行期自研采样仅作轻量增强（PRD §1.3 / M3），不自研重型 CPU 采样分析器替代 spark。

## 备选方案
- **运行期也自研**：被否决，属重复造轮子，spark 已成熟覆盖。
- **启动期也外包（交给 spark）**：被否决，spark 难介入 premain 阶段，覆盖不到本项目首要的启动期场景。
