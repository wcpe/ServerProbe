# 范围纪律（防范围漂移 / 镀金）

> 依据 `docs/PRD.md` 的分期与迭代规划（M1–M4）。**只做当前阶段该做的，不提前做、不顺手做。**
> 本项目已交付 M1 + M2 + M5（启动期 agent 增强），当前活跃前沿为 M3。

## 1. 已交付范围（已落地，勿重复造轮子）

- **启动性能剖析（FR1）**：端到端总时长、逐插件 onEnable、逐世界、生命周期分段、启动画像落盘与上次对比、慢启动告警。
- **可选启动期 premain agent（FR1.7）**：逐插件/库下载/世界/配置/事件/命令精确耗时、多线程折叠栈采样、启动火焰图 + 嵌套时间线（`/probe flamegraph`）、HTTP/TCP 外呼监控（`/probe http`）。**当前仅 1.21.4 Paper 单端真机。**
- **运维指标（FR2.1/2.2/2.3）**：JVM 全套、服务器 TPS/MSPT（多版本兼容 + Folia 全局 N/A）、世界区块/实体（Folia 路线 1）。
- **存储与聚合（FR3）**：内存环形缓冲、本地文件落盘（启动画像 JSON + 指标历史 JSONL）、聚合（TPS 滑窗 / MSPT 分位 / GC 差分）。
- **呈现（FR4.1/4.2/4.4）**：`/probe` 八子命令、Prometheus `/metrics`、历史文件对比。
- **告警（FR5）** / **全版本多平台骨架（FR6）** / **开放接口（FR8：只读 API + 存储 SPI + 静态门面）**。

## 2. 后续阶段（禁提前实现 / 禁镀金）

以下属 M3/M4 或规划中，**当前不做、不留占位字段**：

- **FR4.3 Web 面板**（M3）：启动画像详情、历史趋势、可选火焰图、鉴权 + 绑定地址。
- **FR2.6 运行期 CPU 采样归因**（M3，P2）：`ThreadMXBean` 周期采样按 ClassLoader 归并（注：**启动期**栈采样已由 agent 提供，此处特指**运行期**常态归因）。

- **Folia 完整能力**：per-region TPS/MSPT 明细、`callRegion{}` 实体/方块实体采集（当前 Folia 仅区块数）。
- **FR7 Incision 方法级精确归因**（M4，可选）：须先 PoC，默认关闭。
- **Velocity 代理端**（架构已预留，未实现）。

一旦在代码 / 数据模型 / 契约里看到上述能力的提前实现或占位字段 → **删除，或停下来问**，不得镀金。

## 3. 不为未来预留空壳

- 不写"以后可能用"的抽象、配置项、接口、字段。需要时再加。
- 后续阶段能力到时按域新增包（如 `web/`、`incision/`），当前不留占位。

## 4. 越界先问

- 若某任务看起来需要某个后续阶段能力才能完成 → **停止并向用户确认**，不自行扩大范围。
- 简洁方案优先：实现远多于必要（如 200 行 vs 50 行）时重写。资深工程师会觉得过度复杂的，就是过度。
